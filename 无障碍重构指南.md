## 一、自动化无障碍检查脚本使用指南

### 功能特点
1. **智能排除**：自动识别已适配的控件，避免重复处理
2. **深度扫描**：递归遍历整个视图树，找出所有未适配控件
3. **问题分类**：按严重程度和控件类型分类问题
4. **一键修复**：自动生成并应用修复代码
5. **详细报告**：生成HTML/文本格式的检查报告

### 使用方法

#### 方法1：快速扫描（推荐）
```kotlin
// 在任何Activity中调用
val rootView = window.decorView.findViewById<ViewGroup>(android.R.id.content)
val report = AccessibilityScanner.quickScan(this, rootView)
println(report)
```

#### 方法2：一键修复
```kotlin
val (fixedCount, finalReport) = AccessibilityScanner.quickFix(this, rootView)
println("修复了 $fixedCount 个问题")
println(finalReport)
```

#### 方法3：集成到BaseActivity
```kotlin
// 在BaseActivity中添加：
override fun onWindowFocusChanged(hasFocus: Boolean) {
    super.onWindowFocusChanged(hasFocus)
    if (hasFocus && BuildConfig.DEBUG) {
        runAccessibilityCheck()
    }
}

private fun runAccessibilityCheck() {
    val scanner = AccessibilityScanner(this)
    val rootView = window.decorView.findViewById<ViewGroup>(android.R.id.content)
    val issues = scanner.scanViewTree(rootView)
    
    if (issues.isNotEmpty()) {
        Log.w("Accessibility", "发现 ${issues.size} 个无障碍问题")
        // 可选：自动修复或提示开发者
    }
}
```

### 检查的问题类型
1. **缺少内容描述** - 按钮、图片、输入框等
2. **焦点问题** - 可点击但不可聚焦
3. **无障碍重要性** - 设置为IMPORTANT_FOR_ACCESSIBILITY_NO
4. **特殊控件** - 进度条、列表、开关等需要专门支持
5. **操作支持** - 缺少无障碍操作（点击、长按等）

## 二、语音合成界面重构说明

### 设计理念
1. **空间精简**：用进度条替代传统列表，减少控件数量
2. **操作直观**：进度条更符合语音参数的连续性特点
3. **即时反馈**：试听按钮提供实时效果预览
4. **无障碍友好**：所有控件都有完整无障碍支持

### 界面布局
```
┌─────────────────────────────────┐
│ 语音选择              [×]关闭    │
├─────────────────────────────────┤
│ 语音类型：                       │
│ [◀] ────────────────○────── [▶] │
│ 中文普通话 (中国)                │
│                                  │
│ 音调：1.0                        │
│ ────────────────○────────────── │
│                                  │
│ 语速：1.0                        │
│ ────────────────○────────────── │
│                                  │
│ [ 试听 ]          [ 保存 ]       │
└─────────────────────────────────┘
```

### 使用方式
```kotlin
// 显示语音选择对话框
val dialog = TtsVoiceSelectorDialog.show(ttsEngine)
dialog.show(supportFragmentManager, "tts_voice_selector")

// 或者直接集成到现有TTS设置中
binding.btnTtsSettings.setOnClickListener {
    TtsVoiceSelectorDialog.show(ttsEngine).show(supportFragmentManager, "tts")
}
```

### 无障碍特性
1. **进度条**：支持无障碍调整操作
2. **试听按钮**：详细描述+长按提示
3. **焦点顺序**：合理的Tab键导航顺序
4. **语音反馈**：实时更新内容描述
